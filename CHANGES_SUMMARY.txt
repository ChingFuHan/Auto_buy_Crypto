PROBLEM FIXED
─────────────
When running: python3 main.py backfill

The program would crash with error after hitting Binance API rate limit (429):
  "Too many requests; current limit of IP is 2400 requests per minute"

This happened because the rate limit detection was local to each request,
not persisting across the entire backfill operation.

SOLUTION IMPLEMENTED
────────────────────
Converted rate_limit_wait_until from a LOCAL variable (lost after each 
request) to an INSTANCE variable (persists across all requests).

Now when rate limit is hit:
1. Global wait window is set: rate_limit_wait_until_ms = now + 60_000 ms
2. ALL subsequent requests check this window
3. If still waiting, they sleep until window expires
4. After 60+ seconds, they proceed normally
5. Process continues until all data is downloaded

1. pump_system/exchange/binance_client.py

   ───────────────────────────────────────────────
   self.rate_limit_wait_until_ms = 0.0
   
   
   Change 2: Modified _request() method:
   ───────────────────────────────────────────────
   BEFORE: Local variable
     rate_limit_wait_until = 0.0
     
   AFTER: Instance variable (persists)
     self.rate_limit_wait_until_ms = now_ms + 60_000
     
   Added pre-request check:
   - Before attempting HTTP request, check if global rate limit window active
   - If yes, sleep until it expires
   - This prevents cascading 429 errors from parallel requests

KEY TECHNICAL DETAILS
──────────────────────
Class: BinanceClient
Instance Variable: rate_limit_wait_until_ms

Initialization: 0.0 (no wait)

When 429 error:
  rate_limit_wait_until_ms = now_ms + 60_000

Before each request:
  if rate_limit_wait_until_ms > now_ms:
    wait until window expires
    continue request

After window expires:
  rate_limit_wait_until_ms automatically cleared on next successful request
